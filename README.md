� Finlite: My Personal Finance Dashboard

This is *my* take on a finance app – nothing template‑y, no corporate branding, just code I wrote and features I actually use. I built it because other apps felt like generic clones; this one reflects the way **I** think about money.

The idea is straight‑forward: give yourself a clean ledger, see where your cash is going, and visualize progress toward real goals.

## 🛠 How I Built This
This project serves as my deep dive into the MERN (specifically React/Vite) ecosystem and cloud-based data management.

The Frontend: Built with React and Vite for near-instant load times.

Styling Strategy: I used Tailwind CSS to maintain a clean, "Utility-First" design system. It allowed me to build custom components without writing thousands of lines of CSS.

State & Data: I started with Supabase for real‑time syncing and auth, but you can replace it with any database or even a simple REST API. The code is structured so the data layer is easy to swap out – the values in `.env` are just environment variables.

Visuals: Data visualization is handled by Chart.js, which turns messy transaction arrays into readable spending patterns.

## 🧩 Key Functionality
The Ledger: A simple, intuitive interface for logging daily transactions.

Visual Insights: A dynamic dashboard that breaks down spending by category (Food, Rent, Entertainment, etc.).

Goal Tracking: A "Wise Path" module that helps users calculate how long it will take to reach specific savings milestones.

Responsive Flow: The entire UI is mobile-first, because you’re usually checking your balance while you're out, not just at a desk.

## 🚀 Local Development
The repository is yours—when I open the folder I see my own code, not something generated by Lovable or another tool.

1. **Clone the repo** (or fork it to make it your own):
   ```bash
   git clone https://github.com/Utkarshkumar18/finlite-wise-path.git
   cd finlite-wise-path
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Set up environment variables.**
   - Copy `.env.example` to `.env`, then fill in the values for whatever backend you're using.
   - Example format (Supabase shown):
     ```env
     VITE_SUPABASE_URL=your_supabase_url
     VITE_SUPABASE_ANON_KEY=your_supabase_key
     ```
   - If you host on Vercel, add the same variables in the Vercel dashboard under the *Environment Variables* section.

4. **Run the dev server:**
   ```bash
   npm run dev
   ```

> When you push changes, connect the GitHub repo to Vercel and it will automatically build and deploy. There’s nothing magical – just a standard Vite project.
